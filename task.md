## Fase 5: Algoritmo de Índice y Despliegue
- [x] Programar la lógica del Scoring de inversión en Python y generar la capa consolidada. Implementado en `scripts/process_data.py` (`cargar_denue`, `calcular_indice_comercios`, `calcular_indice_inversion`, `exportar_capa_indice_inversion`). El componente "Comercios" se calcula con el DENUE (escuelas=SCIAN 61, salud=SCIAN 62, supermercados) como cercanía del centroide de cada AGEB al establecimiento más próximo de cada categoría (decaimiento lineal hasta 3 km). **Riesgo reincorporado:** el componente "Riesgo" (peso 0.3) ya se aplica como penalización (hasta −30 pts) usando la exposición a inundación IMPLAN por AGEB (`RIESGO_INDEX`, ver Fase 2). El índice base es Servicios (0.4) + Comercios (0.3) renormalizado a 0-100, menos la penalización, recortado a [0,100]. Verificado (ej.: EL DIVISADERO base 65.5 − 21.2 = 44.3). Salida: `data/indice_inversion.geojson` (340 AGEBs, 506 KB).
- [x] Agregar la capa del "Índice de Inversión" al mapa de Leaflet con una escala de color verde-rojo. Checkbox habilitado, choropleth de 5 escalones (mismos cortes que Servicios), leyenda y ficha de detalle propia (`mostrarDetalleInversion`: desglose de escuelas/salud/supermercados + índice de comercios + índice final). Se generalizó el sistema de leyendas para soportar varias capas activas a la vez (muestra la de activación más reciente). Verificado en navegador con pruebas atómicas (clic, alternado de capas).
- [x] Realizar pruebas locales finales levantando el servidor ligero de Python. Verificado con `python -m http.server`: sin errores de consola ni de red, ambas capas cargan y se pueden alternar, sidebar y leyenda funcionan, escritorio y móvil (375×812) probados. **Bug encontrado y corregido:** en el media query móvil, la regla base `#details-sidebar` traía `transform: translateY(450px)` fijo (no solo en `.sidebar-collapsed`); como un selector de ID tiene más especificidad que uno de clase, esto dejaba el sidebar de detalles permanentemente fuera de pantalla en móvil sin importar su estado. Bug preexistente del scaffold inicial, no introducido en esta sesión. Corregido en `styles.css` quitando esa línea redundante; verificado con ciclo completo cerrado→abierto→cerrado en viewport móvil.
